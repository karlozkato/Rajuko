# Documentación del proyecto RAJUKO

Esta carpeta contiene toda la documentación viva del proyecto. "Viva" significa que no se escribe una vez y se olvida — se actualiza cada vez que se toma una decisión importante, se cierra una etapa, o cambia algo en la visión del negocio.

## Índice de archivos

| Archivo | Para qué sirve | Cuándo actualizarlo |
|---|---|---|
| `vision-del-proyecto.md` | Qué es RAJUKO, por qué existe, hacia dónde va (incluye la visión ampliada de plataforma de identidad) | Cuando cambie el concepto de negocio o se agregue una fase nueva |
| `roadmap.md` | Plan de desarrollo por etapas, con estado actual de cada una | Cada vez que se termina o se empieza una etapa |
| `guia-de-desarrollo.md` | Reglas prácticas de código: estructura de carpetas, convenciones, checklist de SEO, cómo agregar productos/secciones | Cuando se define una nueva convención o patrón de código |
| `identidad-de-marca.md` | Paleta de colores, tipografía, logo/isotipo/imagotipo, tono de voz, inspiración | Cuando cambie cualquier elemento de identidad visual |
| `decisiones-tecnicas.md` | Registro de decisiones técnicas y el porqué de cada una (stack, patrones, efectos) | Cada vez que se decide "cómo" resolver algo técnicamente |
| `reuniones.md` | Bitácora cronológica de sesiones de trabajo: qué se hizo, qué quedó pendiente | Al final de cada sesión de trabajo en el proyecto |

## Cómo usar esta documentación

- Si vas a tomar una decisión de diseño → revisa primero `identidad-de-marca.md`, para no romper consistencia.
- Si vas a escribir código nuevo → revisa `guia-de-desarrollo.md` antes de empezar.
- Si no recuerdas por qué se hizo algo de cierta forma → búscalo en `decisiones-tecnicas.md`.
- Si quieres saber qué sigue → `roadmap.md` tiene el estado real, actualizado, de cada etapa.
- Si necesitas retomar el hilo después de un tiempo sin tocar el proyecto → `reuniones.md` te pone al día rápido.
